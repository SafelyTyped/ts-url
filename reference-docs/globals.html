HRefParts

HRefParts: HRefPartsWithHostname | PRHRefParts | HRefPartsWithPathname | HRefPartsWithSearch | HRefPartsWithHash
  • Defined in src/v1/HRefParts/HRefParts.ts:49

HRefParts holds the parts of a URL, using terms from the WHATWG specification.

HRefParts is designed so that you can't compile a data structure containing unsupported combinations of parts.

validateURLData

  • validateURLData(path: DataPath, input: unknown, __namedParameters?: { base: undefined | string }): AppErrorOr<URL>
    • Defined in src/v1/URL/validateURLData.ts:60

    validateURLData() is a data inspector. Use it to determine whether or not the given input contains a well-formed URL or not.

    No attempt is made to prove that whatever the URL points at exists.

    NOTE: for partial URLS (e.g. fragments), and for relative URLs, you need to set base too. The final URL will be evaluated by combining base and input.

    Parameters

    • path: DataPath

      Where are you in the nested structure that you are trying to validate? Use {@link DEFAULT_DATA_PATH} if you are not inside a nested structure.

    • input: unknown

      The data to validate.

    • Default value __namedParameters: { base: undefined | string } = {}
      • base: undefined | string

        If input is not a full URL, set base to a full URL.

    Returns AppErrorOr<URL>

    • input on success, or
    • an AppError explaining why input failed validation
